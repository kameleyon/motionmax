# C-12-3 (Signal S-C3) refresh — 2026-05-10.
# Authenticated SPA surfaces are explicitly disallowed; public marketing
# routes (Astro `/`, React /pricing/help/terms/privacy/acceptable-use,
# /share/:token public previews) stay allow-listed. Aligned with the
# /app-shell.html `<meta name="robots" content="noindex, nofollow">`
# default so the signal is consistent at both the robots and the HTML
# layer. Stale /workspace and /voice-lab paths kept as legacy disallows
# (no longer routes, but external bookmarks still point at them).
#
# Wave C Signal S-M4 / L-M-11 refresh (2026-05-10):
#   • Removed `Crawl-delay: 10` — a 10-second floor chokes crawl budget
#     on a young domain. Google ignores it anyway; only Bing and Yandex
#     read it. With <100 indexable URLs we want every visit to count.
#   • Unified AI-crawler policy: GPTBot, ChatGPT-User, CCBot,
#     anthropic-ai, Claude-Web, Google-Extended, PerplexityBot all hard-
#     blocked. Position: we do not opt into AI training on our content.
#     Public marketing pages can be summarised by the live-search bots
#     that obey robots — but training datasets do not get crawl rights.
#     See docs/ai-training-policy.md for the full statement.

# Position: we do not opt into AI training on our content. The previous
# policy was internally inconsistent (CCBot hard-blocked, GPTBot/
# Claude-Web only blocked from /app), so an AI bot following robots.txt
# could decide our public copy was fair game while a "stricter" bot
# inferred we'd opt out entirely. Now uniform: every named AI training
# crawler is fully disallowed. Bots that respect robots will skip; bots
# that don't aren't constrained by anything here anyway.
